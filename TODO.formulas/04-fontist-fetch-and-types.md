## fetch-data.sh

```bash
log "copying details/ (optional)"
mkdir -p "$PUBLIC/details"
if [[ -d "$TMP/archive/details" ]]; then
  cp -r "$TMP/archive/details/." "$PUBLIC/details/"
else
  printf '  (no details/ on upstream archive — formula pages degrade to summary)\n'
fi
```

Note: `fontist-archive-public` is what we actually fetch from (the script clones `fontist/fontist-archive.git` — see line 22 of `fetch-data.sh`). Once the public archive has `details/`, this script picks it up.

The fallback `(...)` message matches the existing pattern: missing data degrades gracefully rather than failing the build.

## Loader (`src/lib/formulas/details-loader.ts`)

New file. Mirrors the existing `loader.ts` pattern (single-shot cache per slug).

```ts
import { fetchJson } from '../ssr-fetch'
import type { FormulaDetails } from '../types/domain'

export type { FormulaDetails }

const cache = new Map<string, FormulaDetails | null>()

export async function findFormulaDetails(slug: string): Promise<FormulaDetails | null> {
  if (cache.has(slug)) return cache.get(slug) ?? null
  try {
    const details = await fetchJson<FormulaDetails>(`details/${slug}.json`)
    cache.set(slug, details)
    return details
  } catch (e) {
    cache.set(slug, null)
    return null
  }
}
```

Why per-slug cache instead of one-shot like `loadAllFormulas()`: there are ~4,283 formulas. Loading all details at once is wasteful when a single page only needs one. Cache hits are still O(1) per slug.

Why `null` on miss: lets the page distinguish "tried and missing" from "haven't tried yet" without re-fetching.
