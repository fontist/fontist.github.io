# 14 — Implementation plan: fonts domain, multi-version, multi-style, compare-via-Unicode

Plan date: 2026-06-24. Status: **active execution**. Picks up where `13-font-formula-page-architecture.md` left off — Phase 1 (loader split) and Phase 2 (reverse index) are both in scope here. Phase 3 (URL rename) stays deferred because it requires user sign-off on public URL changes.

This plan absorbs every TODO the user named in chat on 2026-06-23/24:

1. Wire the restored `FontViewer.vue`, `FontSpecimen.vue`, `FontUnicodeBrowser.vue`, `UnicodeBlockGrid.vue`, `ssg-routes.ts` files into live routes (they were moved from `../formulas/` in commit 95d3964 and were never wired up — that is the only reason they appeared "orphaned").
2. Implement multi-version handling: the same family (e.g. Roboto) shipped by multiple formulas (e.g. `google` + `roboto`) must be reachable as one FontFamily with `formula_slugs[]` provenance.
3. Implement multi-style within a family: every style/weight across every providing formula must be addressable in the Unicode browser.
4. Implement compare-via-Unicode-browser: extend `/unicode/block/:slug` to accept multiple fonts and show coverage side-by-side.
5. Build the missing `/fonts` (plural) index page for browsing FontFamily entities.
6. The WOFF specimens in `public/fonts/*.woff` come from `../fontist-archive/` via `scripts/fetch-data.sh` — already wired. The restored components already consume them via `injectFontFace(slug, \`fonts/${slug}.woff\`, redistributable)`.

## Why these matter

The current `/font/:slug` route serves `findFormula(slug)` — it renders a *formula*, not a *font family*. AUDIT A.1 + 13-font-formula-page-architecture.md §"Current state" spell out the consequence: there is no URL for "Roboto" the family when two formulas ship it, and variable-font styles are not addressable. The user's chat demands ("HANDLE MULTIPLE VERSIONS OF THE SAME FONT", "MULTIPLE OF THE SAME FONTS IN THE SAME FAMILY IN THE UNICODE BROWSER") are the user-visible fix for this.

## Architecture decisions locked here

1. **FontFamily is the new primary font entity.** `/fonts/:familySlug` is the canonical URL. The existing `/font/:slug` stays as an alias — it continues to render `FontPage.vue` against `findFormula(slug)` for back-compat. No URL rename this session (Phase 3 of TODO 13).
2. **`font-families.json` is the source of truth for family → formulas reverse index.** Generated by `scripts/gen-font-families.mjs` at build time. Read by `loadFontFamilies()` / `loadFontFamily(slug)` in `src/lib/fonts/loader.ts`.
3. **Family slug = kebab-case of the canonical name** from `fonts.json` (`FontEntry.canonical_name`). Two formulas shipping "Roboto" both land in family `roboto`.
4. **Style enumeration** is derived at runtime from the union of `coverage/*.json` files across all providing formulas. Each coverage file's `variable_axes` + the formula's family style name identifies one style. The browser lets the user switch which file's coverage is rendered.
5. **Compare-via-Unicode** reuses `UnicodeBlockGrid` mode=`multi-font` — already implemented in `src/lib/unicode/components/UnicodeBlockGrid.vue:149-159`. The new route is `/unicode/block/:blockSlug?fonts=a,b,c`.

## File-by-file changes

### New files

#### `scripts/gen-font-families.mjs` (new)
Build-time generator. Reads `public/fonts.json` + `public/font-metadata.json` + `public/formulas-data.json`, groups every `FontEntry` (already deduplicated by `canonical_name` upstream) into a FontFamily record.

Output shape — `public/font-families.json`:
```json
{
  "generated_at": "2026-06-24T...",
  "total_families": 6234,
  "families": [
    {
      "slug": "roboto",
      "name": "Roboto",
      "formula_slugs": ["google", "roboto"],
      "style_count": 18,
      "redistributable": true,
      "license_name": "Apache-2.0",
      "files": [
        { "slug": "google-roboto", "formula_slug": "google", "style": "Regular", "path": "fonts/google-roboto.woff", "redistributable": true }
      ]
    }
  ]
}
```

Wired into `scripts/fetch-data.sh` after the upstream pull. Also added to `package.json` as `gen-font-families` script for manual runs.

#### `src/pages/FontsPage.vue` (new)
Family index. Lists every FontFamily with style count, formula provenance, license. Replaces nothing — new URL `/fonts`. Search + A–Z jump bar mirror the FormulaBrowser pattern.

#### `src/pages/FontFamilyPage.vue` (new)
Family detail page. Renders:
- Header: family name, license, redistributable badge
- **Style selector**: chips for every style across every providing formula. Selecting one updates the specimen + the Unicode browser below.
- **FontSpecimen.vue** wired in for the specimen pane.
- **FontUnicodeBrowser.vue** wired in for the coverage pane. Takes the currently-selected style's `slug` as its `slug` prop.
- **Provenance sidebar**: list of `formula_slugs[]`, each linking to `/formula/:slug`.
- "Compare in Unicode browser →" link to `/unicode/block/basic-latin?fonts=<familySlug>`.

#### `src/lib/fonts/families-loader.ts` (new)
Split from `loader.ts` for MECE. Owns:
- `loadFontFamilies(): Promise<FontFamilyIndex>`
- `loadFontFamily(slug): Promise<FontFamily | null>`
- `findFamilyByFile(fileSlug): Promise<FontFamily | null>` — reverse lookup from a coverage file slug.

`loader.ts` continues to own the raw upstream registries (`loadFontsRegistry`, `loadFontMetadata`).

### Modified files

#### `src/lib/types/domain.ts`
Add the FontFamily entity and supporting types:

```ts
export interface FontFamilyFile {
  slug: string                 // coverage file slug (unique per file)
  formula_slug: string
  style: string                // "Regular", "Bold Italic", etc.
  path: string                 // woff specimen path under public/
  redistributable: boolean
}

export interface FontFamily {
  slug: string                 // canonical family slug
  name: string                 // display name
  formula_slugs: string[]
  style_count: number
  redistributable: boolean
  license_name: string
  files: FontFamilyFile[]
}

export interface FontFamilyIndex {
  generated_at: string
  total_families: number
  families: FontFamily[]
}
```

#### `src/router.ts`
Add new routes — keep all existing `/font/*` routes as back-compat aliases:
- `/fonts` → `FontsPage.vue`
- `/fonts/:familySlug(.+)` → `FontFamilyPage.vue`
- `/fonts/:familySlug(.+)/unicode` → `FontFamilyUnicodePage.vue` (delegates to `FontUnicodeBrowser.vue` with multi-style switcher)

#### `scripts/gen-ssg-routes.mjs`
- Add `/fonts` to `STATIC_ROUTES`.
- After the formula loop, read `public/font-families.json` and add `/fonts/<family.slug>` + `/fonts/<family.slug>/unicode` per family.
- Add to `.gitignore`: `/public/font-families.json`.

#### `src/pages/UnicodeBlockPage.vue`
Accept `?fonts=a,b,c` query. When present, fetch coverage for each and pass N `FontContext` objects to `UnicodeBlockGrid mode="multi-font"`. Each font gets a distinct `color` from a fixed palette so users can tell cells apart.

#### `src/components/FontUnicodeBrowser.vue` (the restored one)
Promote to TypeScript. Add an optional `fileSlugs: string[]` prop (multi-style). When present, render a style-switcher above the block list and re-fetch coverage when the user changes style. Default behavior (single slug) unchanged.

#### `src/components/FontViewer.vue` (the restored one)
The 1173-LOC 3-panel power-user inspector. Wire into `/fonts/:familySlug?view=inspector` — opt-in via query param so the default page stays the marketing specimen. Documented but not required for the family page.

#### `src/components/UnicodeBlockGrid.vue` at `src/components/`
This duplicates `src/lib/unicode/components/UnicodeBlockGrid.vue`. Audit side-by-side:
- Live version (lib/unicode/components/) has typographic guides, control-char boxes, multi-font mode.
- Restored version (src/components/) is simpler — fewer features, smaller surface.

**Decision: delete the duplicate `src/components/UnicodeBlockGrid.vue`.** The live one is strictly richer. The restored file was the formulas-era original that the live one replaced. (This is the *one* file from the restoration that is genuinely redundant; the audit in `feedback-zero-imports-does-not-mean-dead` flagged this exact case as "verify before deleting".)

Will be proposed to the user, NOT deleted unilaterally.

#### `src/ssg-routes.ts` (the restored TS route builder)
Duplicates `scripts/gen-ssg-routes.mjs`. The mjs version is wired into `package.json` build; the ts version is not.

**Decision: delete `src/ssg-routes.ts`.** Same justification as above — the .mjs is canonical and wired. Will be proposed, not deleted unilaterally.

### Phase A — Types + generator + loader (no UI)
1. Add `FontFamily`, `FontFamilyFile`, `FontFamilyIndex` to `src/lib/types/domain.ts`.
2. Write `scripts/gen-font-families.mjs`. Verify it runs against the current `public/*.json` and produces a valid `font-families.json`.

### Phase B — `/fonts` index page

2. Add `/fonts` route. Add to `gen-ssg-routes.mjs STATIC_ROUTES`. Add nav link.

### Phase C — `/fonts/:familySlug` family page
1. Build `src/pages/FontFamilyPage.vue`.

4. Add route + ssg entries.

### Phase D — Multi-style within family
1. Extend `FontUnicodeBrowser.vue` to accept `fileSlugs: string[]` and render a style switcher.

### Phase E — Compare via Unicode browser
1. Extend `UnicodeBlockPage.vue` to read `?fonts=a,b,c` query.
2. Build a `useMultiFontContext` helper that fetches coverage for each slug and returns `FontContext[]`.
3. Pass to `UnicodeBlockGrid mode="multi-font"` (already implemented).
4. Add a "Compare in Unicode browser →" link from `/fonts/:familySlug` and from `/compare`.

### Phase F — Wire FontViewer (optional power-user view)

2. Document the toggle in the family page UI.

### Phase G — Resolve duplicate UnicodeBlockGrid + ssg-routes.ts
1. Present findings to user. **Do not delete unilaterally.**
2. If approved: delete `src/components/UnicodeBlockGrid.vue` and `src/ssg-routes.ts`.
